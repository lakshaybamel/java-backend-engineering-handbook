# OOPs Interview Questions (Java)

---

## 🧱 Basics

**Q1: What is OOP?**  
A: Programming paradigm based on objects and classes.

---

**Q2: What are the 4 pillars of OOP?**  
A: Encapsulation, Inheritance, Polymorphism, Abstraction.

---

**Q3: Difference between class and object?**  
A: Class is blueprint, object is instance.

---

## 🔐 Encapsulation

**Q4: What is encapsulation?**  
A: Wrapping data and methods together with controlled access.

---

**Q5: How is encapsulation achieved?**  
A: Using private variables and public getters/setters.

---

**Q6: Why use encapsulation?**  
A: Data hiding, security, maintainability.

---

## 🔗 Inheritance

**Q7: What is inheritance?**  
A: Acquiring properties of another class.

---

**Q8: Types of inheritance in Java?**  
A: Single, Multilevel, Hierarchical.

---

**Q9: Why multiple inheritance not supported in classes?**  
A: To avoid ambiguity (diamond problem).

---

**Q10: What is super keyword?**  
A: Refers to parent class members.

---

## 🔁 Polymorphism

**Q11: What is polymorphism?**   
A: One method with multiple behaviors.

---

**Q12: Types of polymorphism?**  
A: Compile-time (overloading), runtime (overriding).

---

**Q13: What is method overriding?**  
A: Child class redefining parent method.

---

**Q14: What is dynamic method dispatch?**  
A: Runtime decision of method call.

---

## 🧩 Abstraction

**Q15: What is abstraction?**  
A: Hiding implementation details.

---

**Q16: How is abstraction achieved?**  
A: Abstract classes and interfaces.

---

**Q17: Can abstract class have constructor?**  
A: Yes.

---

## 📦 Interface

**Q18: What is interface?**  
A: Blueprint with abstract methods.

---

**Q19: Can interface have methods with body?**  
A: Yes (default and static methods).

---

**Q20: What is functional interface?**  
A: Interface with one abstract method.

---

## ⚙️ Constructors

**Q21: What is constructor?**  
A: Special method used to initialize objects.

---

**Q22: Can constructor be overloaded?**  
A: Yes.

---

**Q23: Can constructor be inherited?**  
A: No.

---

## 🧠 this & super

**Q24: What is this keyword?**  
A: Refers to current object.

---

**Q25: What is super keyword?**  
A: Refers to parent class.

---

## 📊 Access Modifiers

**Q26: What are access modifiers?**  
A: Keywords that define visibility.

---

**Q27: Most restrictive modifier?**  
A: private.

---

## 📌 Object Class

**Q28: What is Object class?**  
A: Root class of Java.

---

**Q29: equals() vs == ?**  
A: equals() → content, == → reference.

---

**Q30: Why override hashCode()?**  
A: Required for hash-based collections.

---

## 🔒 Immutability

**Q31: What is immutable class?**  
A: Object cannot change after creation.

---

**Q32: Why String is immutable?**  
A: Security, caching, thread safety.

---

## 🧩 SOLID

**Q33: What is SOLID?**  
A: 5 design principles.

---

**Q34: What is SRP?**  
A: One class → one responsibility.

---

**Q35: What is DIP?**  
A: Depend on abstractions, not concrete classes.

---

## 🎯 Advanced

**Q36: Difference between abstract class and interface?**  
A: Abstract class → partial abstraction, Interface → full abstraction.

---

**Q37: What is upcasting?**  
A: Parent reference to child object.

---

**Q38: What is downcasting?**  
A: Converting parent reference back to child.

---

**Q39: What happens if equals() is overridden but not hashCode()?**  
A: Incorrect behavior in HashMap/HashSet.

---

**Q40: Can we achieve multiple inheritance in Java?**  
A: Yes, using interfaces.

---
